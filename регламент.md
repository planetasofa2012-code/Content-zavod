# 📘 Регламент разработки (Лекции 1–2)

> Рабочий документ для ссылки при разработке IT-продуктов.
> Использование: скинуть этот файл в чат Antigravity + написать «Используй этот регламент для разработки».

---

## 🔄 Цикл разработки

| # | Этап | Что делаем |
|---|------|-----------|
| 0 | Исследование | Анализ рынка, конкурентов, потребностей с помощью ИИ |
| 1 | Идея | Подробное описание идеи. **Структурировать, а не упрощать!** |
| 2 | Техническое задание | Формируется нейронкой на основе идеи. Скелет проекта |
| 3 | Архитектура | Структура файлов и папок. Порядок = качество |
| 4 | Дизайн | Визуал, UX, кнопки, цвета. Есть у любого продукта |
| 5 | Запуск локально | Тестируем на своём компьютере |
| 5.1 | Задачи | Чеклист задач для самопроверки нейронки |
| 6 | Тестирование | Ищем неточности и косяки |
| 7 | Деплой | Заливаем на сервер |
| 8 | Дебаггинг | Исправляем баги |
| 9 | Релиз | Продакшн, продукт доступен пользователям |

**Правило**: цикл 0→9→0 никогда не заканчивается.

---

## 📐 Правила работы с идеей

1. Идею описывать **максимально подробно** — нейронка всё упрощает сама
2. Писать: *«Структурируй мою идею, а не изменяй и не упрощай»*
3. Закидывать транскрипцию разговора с клиентом, чертежи, примеры
4. Проверять результат, при необходимости дополнять и возвращаться
5. **Не пропускать этап идеи** — сырая идея = слабый продукт
6. Если идея не раскрылась — не страшно, удалить папку и сделать заново

## 🧩 Скиллы (субагенты)

### Что это
Скиллы — маленькие роботы-помощники, которые помогают главному агенту (Antigravity) выполнять работу лучше и точнее. Не надо полчаса объяснять задачу — агент сам делегирует нужному субагенту.

### Где найти
- **skills.sh** — 84 000+ скиллов
- Разделы: маркетинг, дизайн, копирайтинг, фуллстэк-разработка, SEO и т.д.
- Сортировка по **Trending** — популярные проверены сообществом

### Два режима установки

| Режим | Где живёт | Когда |
|-------|----------|-------|
| **В проект** | Папка `.agents/skills/` в директории проекта | Скиллы под конкретную задачу |
| **Глобально** | На уровне ОС (внутри Antigravity) | Универсальные: find-skills, code-review |

### Как установить
1. Открыть скилл на skills.sh → скопировать команду сверху
2. Вставить в **Терминал** Antigravity
3. Выбрать: **Project** или **Global**
4. Подтвердить установку

### Как удалить
- **Локальный**: удалить папку из файловой директории
- **Глобальный**: команда `npx skills rm <название_скилла>`
- Или спросить Antigravity: *«Какие скиллы установлены глобально? Удали ненужные»*

### Лайфхак: автоподбор скиллов
1. Установить **find-skills** глобально
2. Написать Antigravity: *«Изучи мой проект и подбери нужные скиллы»*
3. Он сам найдёт, предложит и установит

### Отличие от плагинов
- **Плагины** — программные расширения оболочки Antigravity (визуализация, UI)
- **Скиллы** — markdown-файлы с правилами и кодом, на которые агент ссылается как на промпты

## 🔌 MCP (Model Context Protocol)

### Что это
Умный аналог API, созданный Anthropic. Агент подключается ко всем сущностям сервиса **сразу**, без отдельных строк кода для каждого подключения.

### Как работает (пример с CRM)
- **Без MCP**: отдельный код для денег, сотрудников, статусов, сделок → каждый подключается через API вручную
- **С MCP**: один MCP-сервер → агент сам находит нужную сущность → забирает данные в JSON → отдаёт результат

### Важно запомнить
- MCP нужен **только при разработке**, в продакшене работает обычный API
- Антигравити сам решает, какой MCP-сервер использовать (если установлены)
- Ошибок минимум — агент не перепутает данные

### Где найти MCP-серверы
- **context7.com** — основной каталог
- В самом Antigravity: правая панель → MCP Tools

### Как подключить
1. Скинуть ссылку context7.com в Antigravity
2. Написать: *«Помоги подключить все необходимые MCP-серверы»*
3. Он попросит авторизацию / ключ — предоставить
4. Готово, серверы появятся в настройках

### Полезные MCP-серверы

| Сервер | Назначение |
|--------|-----------|
| **Supabase** | База данных — не надо вручную делать миграции |
| **Next.js** | Фронтенд-разработка |
| **Figma** | Дизайн-макеты прямо из Antigravity |
| **Context7** | Универсальная база MCP-серверов |

## ⚙️ Правила работы в Antigravity

1. **Модель**: Claude Opus. Резерв: Gemini 3.1 Pro. Смена через OpenRouter — один запрос
2. **Продукт либо работает, либо нет** — не заморачиваться качеством кода
3. **Accept All** — подтвердить изменения. **Reject All** — откатить
4. **Файлы называть латиницей** (особенно на Windows)
5. **Если ступор** — скриншот + вопрос: «Что мне дальше делать?»
6. **Если зацикливается** — сбросить чат или сменить модель
7. **Терминал привязан к папке проекта**, новое окно — отдельный проект
8. **Open Manager** (правый верхний угол) — параллельная работа над другой задачей в том же проекте
9. **Не закрывать локальный проект** — тогда не нужно перезапускать, агент просто обновляет
10. **Auto-approve команд** — плагин автоподтверждения терминальных команд (по желанию)

### Кастомизация (Settings → Customization → Manage)
- Можно вставить регламент **на уровне мозгов Antigravity**
- Тогда не нужно каждый раз ссылаться на файл — правила работают автоматически
- ⚠️ Добавлять сюда только проверенные, универсальные правила

## 🔒 Безопасность

1. Antigravity не имеет полной автономности — мы задаём параметры
2. Создавать отдельные папки для проектов
3. Глобальный промпт: *«Не удаляй ничего без подтверждения»*
4. Внимательно читать, что он запускает в терминале
5. Бэкапы — клонировать важные файлы
6. GitHub — весь код на сервере
7. Можно развернуть на отдельном компьютере / виртуальной машине

## 🧠 Регламент как живой документ

1. После **каждой лекции**: транскрибировать → извлечь знания → обновить регламент

4. Дополнять своими находками (пример: «Структурируй, а не упрощай» — находка Александра)

## 🐛 Дебаггинг и решение проблем

| Проблема | Решение |
|----------|---------|
| Нейронка зацикливается | Сбросить чат (Ctrl+Alt+Del агенту) или сменить модель |
| Файлы теряются | Проверить файловую директорию, спросить: «Куда ты положил файл?» |
| Бот не запускается | Прописать команду запуска в терминале (`npm run dev`, `start` и т.д.) |
| Интернет пропал / ключи сбились | Перепроверить API-ключи, перезапустить сервисы |
| Результат кривой | Провести аудит: *«Нарисуй текущую архитектуру, покажи, что есть»* |
| Сам не понимаешь, что происходит | Скриншот всего экрана → *«Объясни, что тут к чему»* |
| Не понятно, про что проект | Написать: *«Изучи файловую директорию и расскажи, про что мой продукт»* |
