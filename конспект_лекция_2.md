## 2. Регламент (бывший «тренажёр»)

### Принцип
1. Провели лекцию → транскрибировали → сделали тренажёр/регламент
2. После каждой лекции — новый регламент
3. **Мердж** (объединение): закинуть оба файла → *«Объедини эти два регламента»*
4. Итог: один растущий документ, на который ссылаться при разработке

### Как использовать
- В новом чате: *«Хочу сделать [продукт]. Используй этот регламент для разработки»*
- Перетащить файл в чат или сослаться на него

### Продвинутый вариант
- **Settings → Customization → Manage** — вставить регламент на уровне самой программы
- Тогда правила работают **всегда**, без явной ссылки
- ⚠️ Пока рано — материала мало, лучше подождать до конца обучения

## 3. Скиллы (субагенты)

### Что это
- Маленькие роботы-помощники для главного агента
- Не нужно полчаса объяснять задачу — агент знает, что делать
- Другое название: субагенты
- **Отличие от плагинов**: плагины влияют на оболочку Antigravity (UI), скиллы — это markdown-правила

### Где найти: **skills.sh**
- 84 000+ скиллов (и число растёт)
- Категории: маркетинг, дизайн, копирайтинг, разработка, SEO и т.д.
- Сортировка по **Trending** — популярные проверены сообществом

### Как установить
1. Зайти на skills.sh → найти нужный скилл
2. Скопировать команду в самом верху страницы
3. Открыть **Терминал** в Antigravity
4. Вставить команду → выбрать **Project** или **Global** → подтвердить

### Два режима

| Режим | Описание | Когда |
|-------|---------|-------|
| **В проект** | Появляется в файловой директории (папка `.agents/skills/`) | Скиллы под конкретный проект |
| **Глобально** | Зашивается в Antigravity, не виден в файлах | Универсальные (find-skills, code-review) |

⚠️ С глобальными осторожно — можно забыть, что установлено, и получить странный результат (например, правила для веба мешают разработке мобильного приложения).

### Как удалить
- **Локальный**: просто удалить папку из директории
- **Глобальный**: `npx skills rm <название>` или спросить агента

### Лайфхак: find-skills
1. Установить **find-skills** глобально
2. Написать: *«Изучи проект и подбери нужные скиллы»*
3. Он предложит список → установит одной командой

### Практический пример (Александр)
- Установил find-skills → попросил изучить проект → получил 9 рекомендаций
- Скинул файл с рекомендациями → *«Установи все, только локально»*
- Один Accept — всё установлено за пару минут

### Как скиллы помогают маркетингу
- Установить скилл копирайтинга в проект с готовым продуктом
- Он сам изучит файловую директорию, поймёт сильные/слабые стороны
- Не просто ищет в интернете — работает в контексте вашего продукта

## 4. MCP (Model Context Protocol)

### Что это
- Создан компанией Anthropic (создатели Claude)
- **Умный аналог API** — подключение ко всему сразу
- Будущая инфраструктура всего бизнеса и цифровой среды

### Сравнение API vs MCP (на примере CRM)

**Задача**: ежедневный отчёт из Битрикс24 (деньги, сотрудники, статусы, сделки, увольнения)

**API-подход**:
- К каждой сущности — отдельное подключение
- Для каждого подключения — свой код
- Документация, строчки кода, SQL-запросы
- Долго, сложно, много ошибок

**MCP-подход**:
- Один MCP-сервер подключен ко **всему сразу**
- Агент сам находит нужную сущность
- Оформляет в JSON → отправляет результат
- Молниеносно, без ошибок

### Ключевые моменты
- MCP используется **только при разработке**, не в продакшене
- Сервис должен **поддерживать MCP** (не все пока поддерживают)
- Агент **сам выбирает**, какой MCP-сервер использовать
- Устанавливать максимальное количество → агент сам разберётся

### Где найти: **context7.com**

### Как подключить
1. Открыть Antigravity → написать: *«Помоги установить MCP-серверы для продвинутой разработки»*
2. Или скинуть ссылку context7.com → *«Подключись сюда»*
3. Агент попросит авторизацию / ключ → предоставить
4. Серверы появятся в настройках (MCP Tools)

### Полезные серверы
- **Supabase** — база данных без ручных миграций
- **Next.js** — фронтенд-разработка
- **Figma** — дизайн-макеты
- **Context7** — универсальная база серверов
- **PostgreSQL** — альтернативная БД

### Пример с Supabase
Без MCP: зайти в Supabase → авторизоваться → вручную делать миграции → копировать SQL → вставлять → запускать → повторять
С MCP: одно подключение → агент сам всё делает, больше не нужно заходить в Supabase

## 5. Параллельная работа агентов

- **Open Manager** (правый верхний угол) — открывает второго агента
- Можно работать с предыдущими чатами или текущим проектом
- **Нельзя**: давать похожие задачи (бэкенд + фронтенд) — конфликтуют
- **Можно**: разные задачи (код + документация, код + маркетинг)
- На мощном компьютере (M4, 24GB RAM) — 2-3 окна без проблем

| Совет | Детали |
|-------|--------|
| Периодически просить нарисовать архитектуру | *«Покажи, что у нас сейчас, какая архитектура»* — помогает находить косяки |

| Нейронки не на уровне AGI | Где-то надо подкорректировать — это нормально |

## 8. Ключевые термины

|--------|---------|
| **Скилл / Субагент** | Помощник главного агента с конкретной специализацией |

| **Репозиторий** | Хранилище кода (GitHub) |

| **Context7** | Каталог MCP-серверов |
| **Open Manager** | Второй агент для параллельной работы |
