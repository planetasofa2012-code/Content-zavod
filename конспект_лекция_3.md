## 1. Что такое GitHub

### Простыми словами
- **Социальная сеть для программистов** — вместо фотографий люди выкладывают свои проекты (код)
- Другие могут посмотреть, оценить, поставить звёзды
- Платформа с **открытым кодом** — миллионы готовых проектов в свободном доступе

### Зачем нужен
1. **Хранение кода** — если что-то случится с компьютером, код сохранён в облаке
2. **Версионирование** — хранятся все версии проекта (версия 1, 2, 3…), можно откатиться
3. **Обмен проектами** — делиться по ссылке, проверять код коллег
4. **Деплой на сервер** — безопасная отправка кода через GitHub, а не напрямую
5. **Совместная работа** — несколько разработчиков могут работать над одним проектом

### Версионирование (важно!)
- Каждое обновление = новая версия в GitHub
- Если что-то сломалось → написать в Antigravity: *«Откатись на предыдущую версию»*
- Нет ограничений на количество версий (в отличие от некоторых серверов, где хранится только 5)

## 2. Схема работы: Продукт → GitHub → Сервер

```
┌──────────────┐        ┌──────────────┐        ┌──────────────┐
│   Продукт    │  ───►  │    GitHub     │  ───►  │    Сервер    │
│  (локально)  │        │ (репозиторий) │        │  (продакшн)  │
└──────────────┘        └──────────────┘        └──────────────┘
       ▲                       │                        │
       │                       │                        │
   Разработка            Хранение кода             Пользователи
   в Antigravity         + версии                  имеют доступ
```

### Принцип
1. **Разрабатываем** продукт в Antigravity (работает локально, на вашем компе)
2. **Отправляем** код в репозиторий GitHub (хранение + версии)
3. **Деплоим** из GitHub на сервер (автоматически при обновлении)

### Почему не напрямую на сервер?
- Могут быть ошибки, сервер может лечь
- Через GitHub безопаснее и быстрее
- Можно откатить версию, если что-то сломалось
- Сторонним разработчикам не нужен прямой доступ к серверу

### Обратная связь
- Баги с сервера **не идут обратно** в GitHub
- Ошибки сервера появляются в **терминале** Antigravity (логи)
- GitHub — это **хранилище**, он не связан с ошибками сервера

## 3. Ветки (Branches) — совместная работа

### Как это работает
1. Вы — главный разработчик, у вас **основная ветка** (main)
2. Наняли помощника → создаёте **отдельную ветку** (например, «младший разработчик»)
3. Помощник вносит изменения **в свою ветку**, не трогая основную
4. Вы проверяете его работу → если всё ок, делаете **мердж** (соединение веток)
5. Изменения попадают в основную ветку → автоматом улетают на сервер

### Важно
- **Не работать** в одном проекте напрямую — могут быть конфликты
- Каждый разработчик — в своей ветке
- Вы контролируете, что попадает в основную ветку

## 4. Практика: создание репозитория (пошагово)

### Шаг 1: Создать репозиторий на GitHub
1. Зайти на GitHub → **бургерное меню** → **Repositories**
2. Нажать **New Repository**
3. Назвать по имени проекта
4. Выбрать видимость:
   - **Public** — рекомендуется на данном уровне (проще для обмена)
   - **Private** — требует дополнительных настроек
5. Нажать **Create Repository**

### Шаг 2: Получить API-токен GitHub (делается один раз)
1. Профиль → **Settings** → прокрутить до самого низа
2. **Developer Settings** → **Personal Access Tokens** → **Tokens (classic)**
3. **Generate New Token** → выбрать **Classic**
4. Подтвердить email-верификацию
5. Срок действия: рекомендуется **No expiration** (удобнее, но менее безопасно)
6. Доступы: **проставить все галочки** (полный доступ)
7. Скопировать полученный токен

### Шаг 3: Подключить Antigravity к GitHub
1. В чате Antigravity: *«Вот мой токен GitHub: [токен]. Подключись к GitHub»*
2. Отправить ссылку на репозиторий
3. Antigravity сам создаст `.gitignore` (защита токенов и ключей от утечки)

### Шаг 4: Залить код
- Написать: *«Залей код в GitHub»* или *«Отправь файловую директорию в репозиторий»*
- Antigravity сам отправит все файлы

### Шаг 5: Обновлять
- После доработок: *«Обнови репозиторий»*
- Можно настроить автообновление

## 5. Практика: скачать чужой проект

### Развернуть из GitHub
1. Найти репозиторий (через поиск или получить ссылку)
2. Зелёная кнопка **Code** → скопировать ссылку
3. В Antigravity: *«Разверни код из этого репозитория: [ссылка]»*
4. Antigravity скачает и развернёт проект
5. Если нужны токены / API-ключи — предоставить

### Поиск готовых проектов
- На GitHub — миллионы открытых проектов
- Можно искать через Antigravity: *«Поищи в репозиториях GitHub открытый код [описание]»*
- Или через Perplexity с интеграцией GitHub
- Нашли → скопировать ссылку → развернуть у себя → пересобрать под свои нужды

### Важно
- Если проект public — автор сознательно его открыл, можно пользоваться
- Часто требуются свои API-токены для работы

## 6. Управление репозиториями

### Сменить видимость (Public ↔ Private)
- В репозитории → **Settings** → прокрутить вниз → **Change repository visibility**

### Удалить код из репозитория
- В Antigravity: *«Удали код из репозитория GitHub»*
- Или вручную через настройки GitHub

### Удалить проект полностью
- Удалить папку через файл-менеджер
- Попросить Antigravity: *«Удали проект из GitHub, с сервера и с компьютера»*

## 7. Скилл для Windows: fix-windows-terminal

### Проблема
- На Windows Antigravity иногда зависает при выполнении команд в терминале
- Неправильные команды Linux вместо PowerShell

### Решение (от Радамира)
- Скилл, который **обучает агента** работать правильно в Windows-среде
- Заставляет использовать правильные команды PowerShell
- Есть подскилл: поддержка стандартных Linux-утилит в Windows

### Как установить
- Скопировать команду `npx ...` из чата → вставить в **терминал** Antigravity
- Установить локально (для теста) или глобально

### Результат
- Команды автоматически подтверждаются
- Нет зависаний терминала
- Нет конфликтов Windows/Linux команд

## 10. Ключевые термины

| Термин | Значение |
|--------|----------|
| **GitHub** | Платформа для хранения кода и совместной работы |
| **Репозиторий** | Хранилище кода проекта на GitHub |
| **Версионирование** | Хранение всех версий проекта для возможности отката |
| **Ветка (Branch)** | Отдельная копия проекта для параллельной работы |
| **Мердж (Merge)** | Объединение изменений из разных веток |
| **Деплой** | Отправка проекта на сервер для публичного доступа |
| **.gitignore** | Файл, который скрывает токены и ключи от публикации |
| **Public / Private** | Видимость репозитория: открытый / закрытый |
| **API-токен GitHub** | Ключ доступа для Antigravity к вашему аккаунту GitHub |
| **fix-windows-terminal** | Скилл для исправления проблем терминала на Windows |
